# TESTING.md — Руководство тестировщика

## Beauty Catalog — Telegram Mini App

**Продакшн:** https://beauty-catalog-omega.vercel.app
**Бот:** https://t.me/Pedicure_Manicure_bot
**GitHub:** https://github.com/nastay1mur-web/beauty_master

---

## 1. Стек и архитектура

| Компонент | Детали |
|---|---|
| Frontend | Vanilla HTML + CSS + JS, без фреймворков |
| Хостинг | Vercel (проект `beauty-catalog`) |
| Бот-backend | Vercel Serverless Function `tg-app/api/webhook.js` |
| Хранилище записей | `localStorage` (ключ `beauty_bookings`) |
| Telegram SDK | `https://telegram.org/js/telegram-web-app.js` |

**Файлы приложения:**
```
tg-app/
├── index.html          — HTML-оболочка, подключает SDK и скрипты
├── css/styles.css      — все стили
├── js/data.js          — данные: MASTER, CATEGORIES, SERVICES, generateSlots()
├── js/app.js           — логика: STATE, роутер, 8 экранов
└── api/webhook.js      — serverless: обработка /start и /help от бота
```

---

## 2. Запуск локально (для тестирования в браузере)

Внешних зависимостей нет. Достаточно открыть файл напрямую:

```
Проводник → tg-app/index.html → правый клик → Открыть в браузере
```

Или через любой локальный сервер:
```bash
cd tg-beauty-catalog/tg-app
npx serve .
# Открыть http://localhost:3000
```

**Важно для браузера:** Telegram SDK (`window.Telegram.WebApp`) загружается, но `initData` пустой. Приложение это определяет и показывает fallback-кнопки вместо нативных MainButton/BackButton. Это штатное поведение.

---

## 3. Тестирование в Telegram

1. Открыть бот: https://t.me/Pedicure_Manicure_bot
2. Отправить `/start` — бот ответит приветствием с кнопкой «💅 Перейти в каталог»
3. Нажать кнопку — откроется Mini App
4. Для сброса состояния: в Mini App нажать `⋮` → **Очистить данные** (сброс localStorage)

## 7. localStorage — ключи

| Ключ | Значение | Назначение |
|---|---|---|
| `beauty_onboarded` | `'1'` | Онбординг пройден. Удалить → покажется снова |
| `beauty_offer_shown` | `'1'` | Оффер показан. Удалить → покажется снова |
| `beauty_bookings` | JSON-массив | Все записи клиента |

**Сброс состояния через DevTools (F12 → Console):**
```js
// Сбросить всё
localStorage.clear();

// Только онбординг
localStorage.removeItem('beauty_onboarded');

// Только оффер
localStorage.removeItem('beauty_offer_shown');

// Только записи
localStorage.removeItem('beauty_bookings');
```

## 8. Известные ограничения (не баги)

- **HapticFeedback** — работает только в Telegram на мобильных устройствах. В браузере и десктоп-клиенте молча игнорируется.
- **MainButton / BackButton** — нативные кнопки Telegram видны только внутри Mini App. В браузере показываются fallback-кнопки (`.page-cta-wrap`).
- **Имя пользователя** — в браузере всегда «Клиент» (нет `initDataUnsafe`).
- **Расписание** — генерируется на клиенте через `generateSlots()` при каждом запуске. Реального backend-расписания нет.
- **Воскресенье** — в расписании отсутствует (намеренно, `dayOfWeek === 0` пропускается).
- **Услуга «Нет слотов»** — если `available: false` в `data.js`, услуга в списке отображается серой и некликабельной.
